# 任务 (Task)

根据下方提供的内容链接，产出一份多层次、结构化的摘要。严格遵循“输出格式与要求”中的每一条指令。 

立即使用 `TodoWrite` 创建包含以下项目的任务列表：
- [ ] **获取内容**：根据URL类型选择适当的MCP工具提取文本
- [ ] **生成结构化摘要**：基于内容创建7部分摘要
  - 确保标签和来源一定要有，这个非常重要
- [ ] **保存到flomo**：总结后的内容通过flomo mcp保存 

# 输出格式与要求 (Output Format and Requirements)

1. 一句话核心思想 (Core Idea in One Sentence):
    - 用一句话总结整篇文章最核心的观点或结论。必须精炼，直击要害。

2. 核心论点 (Key Arguments):
    - 以有序列表的形式，列出支撑核心思想的 3-5 个主要论点。
    - 每个论点后，用一小段话（不超过 50 字）简要说明其论证过程或关键证据。
    - 每个论点，如果有对应的图片可以方便用户更好的理解内容，把图片也展示出来

3. 关键信息与数据 (Key Information & Data):
    - 以无序列表的形式，提取文章中所有关键的、作为证据支撑的数据、事实或案例。
    - 如果没有，请明确指出“本文未提供具体的量化数据或案例。”

4. 结论 (Conclusion):
    - 总结文章得出的最终结论或提出的最终解决方案。

5. 关键术语解释 (Explanation of Key Terms):
    - 列出并解释文章中出现的、可能对普通读者造成理解障碍的核心术语。
    - 如果没有，请明确指出“本文无复杂的专业术语。”

6. 标签(tags)
    - flomo 非常善于通过标签进行关联和总结，基于文章内容理由，生成高质量的标签
    - 必须包含标签：文章总结

7. 来源
    - 在文章末尾附上来源链接，方便在必要情况下进一步展开来源的阅读


# 核心原则 (Core Principles)

 - 方便阅读: 输出 markdown 格式，方便论读者可以直观高效的完成阅读
 - 绝对忠实 (Absolute Fidelity): 严禁任何形式的个人解读、评论或信息增删。所有内容必须直接源于原文。
 - 结构至上 (Structure First): 严格按照上述格式输出，不得更改标题或顺序。
