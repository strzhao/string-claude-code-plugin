# Git Tools 插件

智能 Git 提交工具。自动检测 React 代码改动，应用最佳实践优化，生成高质量的提交信息。

## 功能特性

- 🔍 **智能 React 检测**：自动分析代码变更，智能识别 React 相关代码（无硬编码规则）
- ⚡ **自动化优化**：自动应用 React 最佳实践和代码简化优化
- 📝 **高质量提交信息**：自动生成符合约定式提交规范的提交信息
- 🔄 **完整工作流**：从代码分析到提交的完整自动化流程
- 🛡️ **安全可靠**：提供预览和确认机制，防止意外修改
- 🎯 **用户友好**：清晰的进度反馈和交互确认
- 🧠 **代码理解测验**：提交前智能出题，确保开发者理解 AI 生成的代码
- 📋 **任务同步**：提交后自动同步进度到 ai-todo 任务管理（可选，需安装 ai-todo-cli）

## 核心功能

### 智能 Git 提交工作流

插件提供 `/git-commit` 命令，执行以下完整工作流：

1. **Git 状态分析**：检查当前仓库状态，确认有可提交的更改
2. **React 代码检测**：智能分析代码变更，识别 React 相关代码
3. **React 最佳实践优化**：如果检测到 React 代码，调用 "react best practice" skill 进行优化
4. **代码简化优化**：调用 "code simple" skill 进行代码简化
5. **修改内容总结**：汇总所有优化和改动
6. **代码理解测验**：评估改动复杂度，必要时向开发者提问确认理解
7. **Git 提交执行**：生成高质量提交信息并执行 `git commit`
8. **任务同步**：如 ai-todo-cli 可用，自动同步任务进度

## 安装方法

### 方法一：通过插件市场安装（推荐）
1. 确保已安装 Claude Code 插件市场
2. 运行 `/plugins` 查看可用插件
3. 选择安装 `git-tools` 插件

### 方法二：本地安装
1. 复制插件目录到 Claude Code 插件目录：
   ```bash
   cp -r plugins/git-tools ~/.claude/plugins/
   ```
2. 重启 Claude Code
3. 运行 `/skills` 确认插件已加载

## 使用方法

### 基本使用

在需要提交代码时，使用以下命令：

```
/git-commit
```

插件会自动执行完整的工作流程，包括：
- 分析 Git 状态
- 检测 React 代码
- 应用优化建议
- 生成提交信息
- 执行提交

### 工作流程示例

```
Git 提交工作流进度：
- [x] **步骤 1：分析 Git 状态** - 检查是否有待提交的更改
- [x] **步骤 2：检测 React 代码** - 分析改动是否包含 React 相关代码
- [x] **步骤 3：React 最佳实践优化** - 如果检测到 React 代码，应用优化建议
- [x] **步骤 4：代码简化优化** - 应用代码简化建议
- [x] **步骤 5：总结修改内容** - 汇总所有优化和改动
- [x] **步骤 6：代码理解测验** - 评估改动复杂度，必要时向开发者提问
- [x] **步骤 7：执行 Git 提交** - 生成提交信息并提交
- [x] **步骤 8：任务同步** - 同步提交进度到 ai-todo 任务管理系统
```

### React 检测机制

插件使用智能的 React 代码检测，**不依赖硬编码规则**，而是基于以下特征分析：

1. **文件扩展名分析**：`.jsx`、`.tsx` 文件，以及包含 React 代码的 `.js`、`.ts` 文件
2. **导入语句分析**：`import React from 'react'`、React hooks 导入
3. **代码模式分析**：函数/类组件声明、Hooks 使用、JSX 语法
4. **依赖包分析**：`package.json` 中的 React 依赖

3. 仓库已正确初始化

## 错误处理

### 常见问题解决

1. **Git 仓库未初始化**
   - 错误：找不到 `.git` 目录
   - 解决：提示用户初始化仓库或切换到正确的目录

2. **没有更改可提交**
   - 错误：工作区干净，无修改
   - 解决：提示用户先进行代码修改

3. **优化 skill 不可用**
   - 错误："react best practice" 或 "code simple" skill 调用失败
   - 解决：跳过优化步骤，记录警告信息

4. **Git 操作失败**
   - 错误：权限问题、配置问题等
   - 解决：提供具体的错误解决方案

### 安全注意事项

1. **Git 操作安全**
   - 避免使用 `git add -A` 添加敏感文件
   - 建议选择性添加文件
   - 检查是否包含敏感文件（`.env`、`credentials.json` 等）

2. **代码备份**
   - 应用优化前提供预览
   - 确保用户了解改动内容
   - 提供撤销优化的指导
