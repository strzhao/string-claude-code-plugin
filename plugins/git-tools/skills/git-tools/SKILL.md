# Git 智能提交工作流

这是一个高效的 Git 提交工具，在提交前自动分析代码改动，调用相关优化技能，并生成高质量的提交信息。

## 核心原则

**信任 AI 的智能判断**：您已经具备强大的代码分析能力，本工具旨在提供框架而非限制您的判断。

**高自由度设计**：React 检测、优化应用、提交信息生成都依赖上下文判断，您可以根据具体情况选择最佳方法。

**简洁高效**：只提供必要的工作框架，避免过度指导。

## 工作框架

创建一个简洁的任务列表来跟踪进度：

```
Git 提交工作流：
- [ ] **分析 Git 状态**：检查是否有可提交的更改
- [ ] **智能检测 React 代码**：分析改动是否涉及 React
- [ ] **条件性优化**：根据需要调用优化技能
- [ ] **总结修改内容**：汇总所有优化和改动
- [ ] **代码理解测验**：评估改动复杂度，必要时向用户提问
- [ ] **执行智能提交**：生成提交信息并提交
```

## 关键决策点

### 1. React 代码检测

**基本原则**：
- **自主判断**：基于代码特征而非硬编码规则进行分析

### 2. 优化技能调用

**基本原则**：
- **条件性调用**：根据代码分析结果决定是否调用优化技能
- **透明预览**：在应用优化前向用户展示建议的改动
- **用户确认**：获得用户同意后再应用修改

**可用技能**：
- "react best practice"：React 最佳实践优化（条件性调用）
- "code simple"：代码简化优化（建议总是调用）

**调用方式**：
- 使用 Skill 工具调用相关技能（例如：`Skill: "react best practice"`）
- 提供必要的文件路径和代码片段作为上下文
- 在应用优化前向用户展示建议的改动
- 获得用户确认后使用 Edit 工具应用优化
- 如技能调用失败，记录警告信息并跳过优化步骤（优雅降级）

### 3. 提交信息生成

**语言**：全部使用中文（type 标签除外）

**格式**：`type(scope): 业务描述 (技术说明)`

- **业务描述**：用用户/产品视角说"发生了什么"，简洁优先，一眼看懂
- **技术说明**：用括号补充技术实现，可省略

**type 选取**：
- `feat` 新功能 / `fix` 修复 / `perf` 性能 / `refactor` 重构
- `style` 样式 / `docs` 文档 / `chore` 杂项 / `test` 测试

**示例**：
```
feat(报告): 支持一键导出 PDF (新增导出 API + 前端按钮)
fix(登录): 修复登录后页面空白 (useEffect 缺少依赖导致重渲染)
perf(列表): 长列表滚动更流畅 (虚拟化渲染，DOM 节点从 500→30)
refactor(Auth): 简化鉴权逻辑 (合并重复的 token 校验分支)
chore: 升级依赖版本
```

**禁止**：
- 禁止英文描述（type 标签除外）
- 禁止堆砌技术词汇而忽略业务含义
- 禁止模糊措辞，如「优化代码」「更新逻辑」

### 4. 代码理解测验

**目的**：AI 生成的代码让人对代码库越来越陌生。提交前确认开发者理解即将提交的内容，与工程同步成长。

**基本原则**：
- **智能跳过**：简单改动（格式调整、变量重命名、配置修改、注释更新、依赖版本更新）直接跳过
- **自主出题**：基于本次改动的实际代码，判断开发者应该掌握的知识点
- **聚焦关键**：只问真正重要的——架构决策、业务逻辑、潜在风险、设计模式

**出题指南**：
- 使用 `AskUserQuestion` 工具，1-3 道选择题
- 问题必须来自本次改动的实际代码，不是通用知识
- 错误选项应是合理的误解，而非明显错误
- 示例方向：「这段新增的 useEffect 为什么依赖 X？」「这个 API 变更对下游的影响是什么？」

**答错处理**：
- 解释正确答案及原因，引用具体代码
- 给用户选择：重新作答 或 确认已理解后继续提交
- 目的是学习，不是考试——不强制阻断
